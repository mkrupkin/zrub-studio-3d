# site-design-3d — ЗРУБ.студія

**Іммерсивний 3D-сайт** майстерні зрубів: користувач летить камерою крізь карпатський 3D-світ (гори, ліс, стежка, кабіни), а на «станції конфігуратора» наживо проєктує власний зруб. Скрол керує кінематографічною подорожжю камери. Не 2D-сторінка з віджетом — **весь сайт живе у 3D-просторі**.

Мова інтерфейсу — українська. Референс візуалу — фото Карпат (7cars.com.ua Carpathian-mountains.jpg).

---

## Стек

- **Чистий HTML/CSS/JS**, без білду, без npm
- **Three.js r160** через CDN import map (jsdelivr), ES-модулі
- **HDRI + PBR + пост-обробка** — Poly Haven CC0 асети (CORS-надійні), з graceful fallback
- Шрифти: Fraunces (display) + Karla (body)
- Сервер: `python -m http.server 8123` (НЕ `file://` — ES-модулі)

## Архітектура — ГІБРИД (фото-фон + 3D-передній план)

**Це гібрид**: реальне фото Карпат дає фотореалістичні гори/небо/далекі піки, а 3D — лише передній план (полонина + кабіни + смереки), скомпонований зверху. Так фотореалізм далини гарантований, а інтерактив (конфігуратор зруба) — у 3D. Обрано користувачем як напрям для виставки.

`createWorld(canvas, {onStats, onReady})` → `{setScroll, updateConfigCabin, dispose}`.

**Фото-backdrop:** `assets/backdrop.jpg` (карпатське фото) на великій площині (`940×472`, `MeshBasicMaterial` unlit, `fog:false`, `toneMapped:false` — фото лишається crisp/точним). Позиція `(0, 96, -210)` + друга ширша копія позаду для периферії. `scene.background` = колір неба фото (заповнює краї). ⚠️ **Фото — стокове; для реальної виставки замінити на ліцензоване/власне** (`assets/backdrop.jpg`, будь-яке 2:1 карпатське фото).

**Рельєф (3D):** проста **полога зелена полонина** — FBM малої амплітуди (`terrainH`, ~9м), спадає до −Z (`smooth(-30,-150,z)`), мікро-тераси. Гори НЕ процедурні (їх дає фото). PBR-трава (Poly Haven `aerial_grass_rock`).

**Ліс:** реалістичні смереки — **billboard-хрести** (`makeSpruceTexture`, canvas), InstancedMesh ~550, розміщені ЗА кабінами (z −46..−142, мід-дистанція) — обрамляють і ховають шов між 3D-полониною і фото.

**Деталі:** в'юнка **стежка** (ribbon-сплайн), кабіни, копиці, вогнище.

**Вимкнено в гібриді:** `ridgeRing`/`peakRing` (процедурні гори), `makeCloudTexture`/clouds, `distantHouse` — усе це дає фото. Функції лишились у коді (`void ...`) на випадок повернення до pure-3D.

**Освітлення:** directional «сонце» (сильне, тіні PCFSoft) + hemisphere `~0.38` (щоб тіні не чорніли) + HDRI IBL. Підібрано під освітлення фото (яскравий день). ACES + легкий bloom.

**Камера:** CatmullRom 6 keyframes, скрол 0..1. Дивиться на фото-backdrop (−Z) з полониною+кабінами спереду. Герой → craft → **станція конфігуратора** (кабіна крупно, фото позаду) → галерея → вогнище. ⚠️ Камера рухається ПОМІРНО — плоский backdrop працює під погляд ~−Z; великі повороти вбік показали б край площини (тому вона широка + є друга копія).

## Конфігуратор

- Кабіна на позиції `CFGP` (32,-4) у долині; `updateConfigCabin(cfg)` перебудовує її наживо.
- **Камера станції (keyframe 2,3):** у долині на +X, рівень кабіни `(52, ground+8, 15)` look `(31,4,-6)` — щоб кабіна була ПОВНІСТЮ видна, освітлена, по центру (панель праворуч не перекриває).
- ⚠️ **Гоча:** якщо камеру поставити на схил (високий z) або з низьким hemi — кабіна тоне в тіні/чорноті. Тримати камеру в долині (низький z, плоско) + hemi ≥0.35.
- Кабіна повільно авто-обертається на станції.
- `main.js` мапить слайдери/swatches/дах → `updateConfigCabin`, `onStats` пише площу/колоди/обʼєм/ціну.

## ⚠️ Про реалізм (важливо для очікувань)

Це **процедурний real-time 3D у браузері**. Він досягає гарного стилізованого-реалістичного вигляду (HDRI-світло, PBR-трава, billboard-ліс, атмосферна перспектива, пост-обробка), але **не буде фотореалістичним 1:1 з фотографією** — billboard-дерева пласкі зблизька, рельєф без справжньої мікро-геометрії, матеріали спрощені. Це фундаментальне обмеження підходу.

**Якщо потрібен справжній фотореалізм** (для виставки тощо) — надійний шлях це **гібрид**: реальне фото Карпат як фон-небо/далекі гори (гарантований фотореалізм горизонту) + 3D-передній план (кабіни, стежка) скомпонований зверху. Це вбиває «ігровий» вигляд далини. Обговорити з користувачем перед перемиканням підходу.

Важелі реалізму в поточному 3D (вже застосовані / можна крутити далі):
- Контраст сонце/тінь (сильне directional + помірний ambient) — НЕ занижувати hemi <0.3 (чорніє)
- Соковита насичена зелень + мікро-деталі рельєфу
- Більше шарів хребтів з сильнішим серпанком = глибина
- Щільніші/варіативніші хмари; DoF; кольорокорекція
